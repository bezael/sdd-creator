# Reference — Dominicode voice & project links

## Voice rules

The output is **Spanish**. Match the tone of the existing `tweet.md` and `README.es.md`: direct, technical, anti-hype.

- **Frases cortas.** Una idea por línea.
- **Sin relleno de marketing.** Nada de "estamos emocionados de anunciar", "revolucionario", "game-changer".
- **Habla de lo que gana quien lee**, no de lo que tú construiste.
- **Lema recurrente:** "Spec first. Code after." / "Spec → Plan → Tests → Código." Úsalo como cierre cuando encaje, no en todos los mensajes.
- **Emojis con cuentagotas:** como mucho los de enlaces (📦 🔗 📘 🎓) o un 👇 de transición. Nada de filas de emojis.
- **Primera persona** (Bezael publica en su nombre): "acabo de publicar…", "saqué…".
- **Sin promesas que el changelog no respalde.** Si no está en el CHANGELOG, no se dice.

### Which links go where

- **Tweet "directo" / "problema":** ningún enlace (o solo el repo si sobra espacio).
- **Tweet "con recursos":** instalar + repo (y opcional 1 de libro/curso).
- **LinkedIn:** instalar + repo; libro/curso solo si el usuario lo pide.
- **Hilo:** los enlaces van en el último tweet.
